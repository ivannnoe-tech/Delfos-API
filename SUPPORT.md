# Support — delfos-api

Use este documento para registrar problemas, pedir ajuda e separar suporte de desenvolvimento.

## Canais

1. Issue no GitHub usando o template adequado.
2. Comentário no Pull Request relacionado.
3. Canal interno da equipe quando houver bloqueio operacional.

Para vulnerabilidade, não abra issue pública. Siga `SECURITY.md`.

## Tipos de solicitação

- **Bug**: algo existente falhou ou se comportou diferente do esperado.
- **Feature**: nova capacidade aprovada ou proposta de melhoria.
- **Task**: documentação, refatoração, CI, build, testes ou governança.

## Antes de abrir solicitação

Confira `README.md`, `AGENTS.md`, `DESIGN.md`, `docs/` e issues/PRs recentes.

## Dados sensíveis

Não envie tokens, senhas, chaves privadas, dados reais de cliente ou prints com informação sensível. Use exemplos fictícios e logs sanitizados.
